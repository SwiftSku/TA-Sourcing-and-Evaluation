## CE Spawn Template

Spawn with `model: "sonnet"`. Sequential only — one CE at a time.

### Prompt

```
You are a single-candidate evaluator. Read the evaluation framework at:
[FULL PATH to [active JD file]]

Also read LinkedIn interface learnings at:
[FULL PATH to REF--LIR_Interface_Learnings.md]

Also read anti-detection rules at:
[FULL PATH to REF--Anti_Detection.md]

Evaluate this ONE candidate:
- Profile URL or identifier: {profile_url}
- Source: {source_name}
- DELAY_SECONDS: {delay_seconds}
- NEXT_URL: {next_url_or_empty}

Write the result row to:
[FULL PATH to output file from JD config]

After writing the row, follow the anti-detection idle procedure
(REF--Anti_Detection.md §3) for DELAY_SECONDS, then navigate
to NEXT_URL (or close tab if empty).

Return ONLY this summary line:
{Full Name} | {Tier} | {Score%} | {Verdict} | {Current Company}
```

### Inputs (spawner fills these in)

| Input              | Description                                               |
|--------------------|-----------------------------------------------------------|
| profile_url        | LIR profile URL or public LinkedIn URL                    |
| source_name        | Source column value (e.g., "ACM LIR Search v5")           |
| delay_seconds      | Random 45-200, never same twice in a row (spawner picks)  |
| next_url_or_empty  | Next candidate's URL, or empty if last in batch           |

### Expected Return

```
{Full Name} | {Tier} | {Score%} | {Verdict} | {Current Company}
```
